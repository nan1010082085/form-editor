# E2.5 容器嵌套策略决策

> 日期：2026-07-20 · 状态：已决策

## 决策

**有限放开：最多 2 层容器嵌套**（根级容器 → 一级子容器）。

不放开任意深度嵌套；不保持「完全单层」现状（当前碰撞检测已支持深层收集，产品能力应与实现能力对齐）。

## 依据

| 维度 | 判断 |
|------|------|
| 用户场景 | 表单内嵌 Card、Tab 内嵌 Form 是高频；3 层以上极少 |
| 技术成本 | `collectAllContainers` 已递归；限制深度可避免 reparent 复杂度爆炸 |
| 性能 | 深度 ≤ 2 时碰撞检测可缓存；更深需专项优化 |
| 产品定位 | 大屏以根级绝对定位为主，嵌套容器主要用于表单/页面分区 |

## 实现约束

1. 拖入时：若目标容器已是「子容器」（父级也是容器），拒绝再放入容器类型，提示「最多嵌套 2 层」
2. 粘贴/模板应用：超出 2 层时自动提升到允许层级
3. 现有深层 Schema 兼容加载，编辑时引导用户扁平化

## 不做

- 本期不做无限嵌套
- 本期不重写碰撞算法（沿用现有递归 + 面积优先）

## 后续

若埋点显示「嵌套失败」投诉升高，再评估放宽到 3 层。
